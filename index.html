Hi! I'm Steven! A Computer Engineer!

I graduated from the University of Florida in 2024 with a bachelor's degree in Computer Engineering. I aim to understand computer systems, how they work, and how to solve problems using them.

My most impactful—and first—experience with computers was sticking coins inside my mom's iMac CD drive, thinking they would shoot out like a slot machine.
The technician had to explain to 5-year-old me that they would not, in fact, shoot out like a slot machine...

Ever since then, I've been fascinated with the inner workings of computers and how they function.

The second most impactful experience came when I took a C++ course in high school. My professor explained that C and C++ were commonly used to control nearly every aspect of a computer's inner workings—from the software used to compile our code down to the BIOS chip that initializes during boot.

I've always been obsessed with the lower layers of a computer system—particularly how the various peripherals inside a computer communicate with each other.

For my senior project, I got to explore this further as my team and I attempted to implement an NES on an FPGA, a project intended for a video game museum.

Despite that experience, I still have a hunger for knowledge about embedded systems and their components. These projects serve as an attempt to satisfy that curiosity—and perhaps some other interests too, like signals and PCB design.

Top Projects

Verilog

My top project using Verilog is a PWM controller.

The design operates with an 8-bit counter that toggles between "1" and "0" using a 2-input multiplexer. To change the PWM output frequency, you can latch a new value into the counter and reset the device for the changes to take effect.

VHDL

My top project in VHDL—and the one I'm most proud of—is a 32-bit MIPS-based multi-cycle CPU.

It features memory-mapped I/O with custom memory decoding logic, supports jumping, branching, multiplication, and logical operations, and includes its own built-in RAM—no external RAM required!

C

My top project in C is my custom system call library.

One of the most fascinating aspects of computers is how we interact with operating systems. But how does a program gain access to resources through the operating system? To explore this, I wrote my own system call library in Linux to better understand this interaction.

C++

What’s better than making any device run DOOM? Making DOOM’s file format mountable as a filesystem in Linux, of course!

This C++ project uses the FUSE library in Linux, along with my custom library, to treat DOOM WAD (Where’s All the Data?) files as their own file directories. This allows you to send a WAD file to anyone, and they can mount it to perform read, write, copy, and other file operations.

Now that’s portability!
