Hi! I'm Steven! A Computer Engineer!

I graduated from the University of Florida in 2024 with a bachelors degree in Computer Engineering. I seek to understand computer systems, how they work, and how to solve problems with them.

My most impactful, and first, experience with computers was sticking coins inside of my moms iMac cd drive, thinking they would shoot out like a slot machine.
The technician had to explain to 5 year old me that they would not in fact shoot out like a slot machine...

Ever since then, ive been fascinated with the inner workings of computers and how they work.

The second most impactful experience was when I took a C++ course in high school, my professor explained to me that C and C++ were commonly used to control basically every inner working of a computer. From the software we use to compile our code, down to the bios chip that gets kicked on during boot.

I've always had an obsession with the lower layers of a computer system. In particular, how the various peripherals inside a computer communicate with eachother.

For my senior project, I got to partially understand this as we attempted to implement an NES onto an FPGA to be given to a video game museum.

But there is still this hunger for knowledge about embedded components that I seek to satisfy, these projects serve as an attempt to do this. And maybe satisfy some other hungers as well. Such as for signals, and PCB design.

Top Projects

Verilog

My top project in terms of verilog is a PWM controller. This design works by having an 8 bit counter switch between "1" and "0" via a 2 input mux. If you need to change the frequency at which the PWM outputs, you can latch a new value into the counter and reset the device for it to take effect.

VHDL

My top project in terms of VHDL, and the one im most proud of for all these projects, is a 32 bit MIPS based multi-cycle CPU. It has memory mapped IO via its own custom memory decoding logic, it can jump, branch, multiply, perform logical operations, and features its own built in ram. No external ram necessary!

C

My top project for C is my very own system call library.

One of the most interesting things about computers is our use of operating systems. But how does a program get access to resources through the operating system? I seek to understand this by writing my own system calls library in linux.

C++

Whats better than making any device run DOOM? Making it so DOOM's file format can be mounted as a file system in linux of course!

This C++ project uses the FUSE library in linux, and my own custom library, to treat DOOM WAD(Wheres All the Data?) files as their own file directories. This way, you can send a WAD file to anyone, and they can mount it, and perform read, write, copy, and other file operations on it.

Now thats portability!

This specific PCB youre looking at runs an ARM based STM32F072 microcontroller.
